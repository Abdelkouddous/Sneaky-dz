# Sneaky DZ — متجر الأحذية الرياضية الفاخرة (Monorepo)

مرحباً بك في **Sneaky DZ**، وهو تطبيق ويب متميز لعرض واقتناء مجموعات الأحذية الرياضية العصرية. تم تنظيم هذا المشروع كـ **مستودع متعدد المشاريع (Monorepo)** يضمن فصلاً تاماً بين واجهة المستخدم الخاصة بالعميل (Frontend) وخادم بوابة البيانات والتحكم الخلفي (Backend).

---

## 📂 بنية مجلدات مساحة العمل (Workspace Structure)

تم تنظيم المشروع في مجلدين أساسيين لضمان إمكانية التوسع البرمجي وسهولة النشر والتشغيل:

```
Sneaky-Dz/ (المجلد الرئيسي)
├── package.json            # منسق مساحة العمل (يوجه أوامر التشغيل للمجلدات الفرعية)
├── README.md               # التوثيق والتعليمات باللغة الإنجليزية
├── README_AR.md            # التوثيق والتعليمات باللغة العربية (RTL)
├── frontend/               # تطبيق العميل وواجهة المستخدم (Vanilla JS / Vite)
│   ├── index.html          # مدخل صفحة الويب الأساسي
│   ├── vite.config.ts      # إعدادات وتشغيل Vite
│   ├── package.json        # حزم واعتمادات الواجهة الأمامية
│   ├── css/                # نظام تنسيق هيكلي مقسم
│   │   └── main.css        # سجل استيراد ملفات التنسيق الكلي
│   ├── js/                 # المنطق البرمجي للجافا سكريبت
│   │   ├── app.js          # الملف الرئيسي والمنسق العام للواجهات
│   │   ├── core/           # النواة البرمجية للتطبيق (الموجه ومدير الحالة)
│   │   │   ├── router.js   # موجه مسارات مخصص يعتمد على الهاش
│   │   │   └── store.js    # مخزن تفاعلي لإدارة حالة التطبيق (Pub/Sub)
│   │   ├── data/           # كتالوج البيانات المحلي (products.js)
│   │   └── pages/          # وحدات التحكم بالصفحات (الرئيسية، المتجر، العربة، لوحة التحكم)
│   └── assets/             # الصور والخطوط والموارد البصرية
└── backend/                # طبقة التحكم الخلفية والخادم (Node.js / Express - للتطوير المستقبلي)
    ├── server.js           # مدخل تشغيل الخادم والـ API
    ├── package.json        # حزم واعتمادات الخادم الخلفي
    └── .env.example        # قالب المتغيرات البيئية للخادم
```

## 🛠️ البنية الهندسية والتقنيات المستخدمة (الواجهة الأمامية)

تم بناء الواجهة الأمامية بالكامل باستخدام لغة جافا سكريبت النقية (Vanilla JS)، وتنسيقات CSS الهيكلية، وأداة Vite لتسريع عملية التطوير والبناء.

### 1. موجه مسارات مخصص يعتمد على الهاش (Custom Hash-Based Router)
*   **الموقع:** [router.js](file:///Users/morsistoredz/antigravity/Dzair-Craft/frontend/js/core/router.js)
*   **النمط البرمجي:** موجه المسار الأمامي (Front-Controller Router).
*   **خيارات التصميم:** تقوم فئة [Router](file:///Users/morsistoredz/antigravity/Dzair-Craft/frontend/js/core/router.js#L1) بالتحكم في عملية التنقل بين الواجهات بشكل حيوي:
    *   تنظيف محتويات الصفحة النشطة لتفادي مشاكل تسريب الذاكرة.
    *   إدراج هياكل التحميل المؤقتة (Loading Skeletons) لتحسين سرعة ظهور المحتوى البصري (LCP).
    *   استدعاء ومعالجة كود الصفحات وعرضها بشكل غير متزامن.

### 2. إدارة الحالة بنمط النشر والاشتراك (Pub/Sub State Store)
*   **الموقع:** [store.js](file:///Users/morsistoredz/antigravity/Dzair-Craft/frontend/js/core/store.js)
*   **النمط البرمجي:** نمط النشر والاشتراك الأحادي (Publisher-Subscriber Singleton).
*   **خيارات التصميم:** تدير فئة [Store](file:///Users/morsistoredz/antigravity/Dzair-Craft/frontend/js/core/store.js#L3) العامة البيانات التفاعلية المشتركة (عدد عناصر العربة، الإجمالي المالي، قائمة المفضلة، تفاصيل تسجيل الدخول).
    *   تشترك المكونات في أحداث تعديل الحالة (مثل: `store.subscribe('cart', cb)`).
    *   تحديث الأجزاء المتأثرة فقط في شجرة الـ DOM دون الحاجة لإعادة تحميل الصفحة كاملة.
    *   حفظ وتخزين الحالة تلقائياً في المتصفح عبر خاصية `localStorage`.

## 🚀 خطوات التشغيل والإعداد المحلي

### متطلبات التشغيل الأساسية: وجود تثبيت لبيئة Node.js (v18+)

يمكنك تشغيل الأوامر مباشرة من المجلد الرئيسي للمستودع؛ حيث سيتم توجيهها تلقائياً للمجلد المناسب بفضل استخدام الرمز المساعد `--prefix`.

1.  **ثبّت حزم واعتمادات مشروع الواجهة الأمامية:**
    ```bash
    npm install --prefix frontend
    ```
2.  **شغّل خادم التطوير المحلي للواجهة الأمامية:**
    ```bash
    npm run dev
    ```
    سيتم تشغيل خادم Vite المحلي. افتح الرابط التالي للمعاينة: `http://localhost:3000`

3.  **قم ببناء نسخة الإنتاج النهائية والمحسنة للواجهة الأمامية:**
    ```bash
    npm run build
    ```

4.  **لتشغيل خادم التحكم الخلفي (للتطوير المستقبلي):**
    ```bash
    cd backend
    npm install
    npm run dev
    ```
